# Hideout Avatar Customization Plan

## 1. 목표

아지트 안의 사람 아바타는 단순한 기본 캐릭터가 아니라,

`각자 취향과 분위기를 반영한 나만의 캐릭터`

가 되어야 합니다.

이번 아바타는 전투형 게임 캐릭터가 아니라
힐링 공간 안을 돌아다니는 생활형 아바타이기 때문에,

- 디테일은 귀엽고 단순해야 하고
- 실루엣이 잘 읽혀야 하고
- 방 분위기와 잘 어울려야 합니다

## 2. 추천 커스터마이즈 구조

복잡한 RPG식 수치가 아니라 슬롯 기반이 가장 적합합니다.

권장 슬롯:

- 피부 톤
- 헤어 스타일
- 헤어 컬러
- 눈/표정
- 상의
- 하의
- 신발
- 소품
- 방 안 행동 스타일

### 소품 예시

- 머리띠
- 안경
- 헤드폰
- 책
- 작은 인형
- 머그컵

## 3. 추천 디자인 원칙

### 1. 큰 실루엣 차이

작은 모바일 화면에서는 세세한 문양보다 실루엣이 더 중요합니다.

그래서 먼저 차이를 크게 주어야 할 영역:

- 머리 모양
- 옷 실루엣
- 소품 유무

### 2. 색은 과하지 않게

아지트 전체가 힐링 톤이라면 아바타도 방과 충돌하면 안 됩니다.

권장:

- 저채도 파스텔
- 크림, 세이지, 머스타드, 코랄, 잔잔한 블루

### 3. 표정은 과장보다 따뜻함

이 아바타는 액션 게임 캐릭터가 아니므로
표정도 생활형이 어울립니다.

권장 표정:

- 기본 미소
- 편안함
- 졸림
- 집중
- 기분 좋음

## 4. 기능 구조

아바타 커스터마이즈는 두 레이어로 나누는 것이 좋습니다.

### A. 외형 커스터마이즈

- 몸, 머리, 옷, 소품 변경

### B. 분위기 커스터마이즈

- 걷는 속도 톤
- 대기 모션
- 상호작용 포즈
- 방 안에서의 감정 표현

예:

- 차분형: 천천히 걷고 작게 손 흔듦
- 활발형: 조금 빠르게 걷고 반응이 큼
- 몽글형: 멈춰 있을 때 가볍게 흔들림

## 5. UI 제안

권장 진입 방식:

- 아지트 안의 거울
- 옷장
- 화장대

즉 메뉴 버튼보다
방 안 오브젝트를 통해 커스터마이즈에 들어가는 것이 더 자연스럽습니다.

### 예시

- 거울 클릭 -> 아바타 꾸미기 패널
- 옷장 클릭 -> 의상 변경 패널

## 6. 저장 모델 제안

```ts
type HideoutAvatarLook = {
  skinToneId: string;
  hairStyleId: string;
  hairColorId: string;
  eyeStyleId: string;
  topId: string;
  bottomId: string;
  shoesId: string;
  accessoryIds: string[];
  vibeId: 'calm' | 'bright' | 'soft' | 'sleepy';
};
```

중요:

- 실제 모델/메시는 슬롯 조합으로 만든다
- 사용자가 하나씩 바꿀 수 있게 한다
- 나중에 아이템 확장이 쉽다

## 7. 단계별 구현 순서

1차:

- 헤어 4종
- 상의 6종
- 하의 4종
- 피부 톤 4종
- 신발 3종
- 소품 3종

2차:

- 표정 변경
- 대기 모션 변경
- 소품 손에 들기

3차:

- 계절 의상
- 친구/가족 아지트별 복장 프리셋

## 8. 지금 코드에 바로 붙이려면

현재 3D 아지트 백본에서는 아바타가 단순한 기본 메시에 가깝습니다.

다음 단계는:

1. 아바타 상태 타입 추가
2. 외형 프리셋 데이터 추가
3. 거울 또는 옷장 오브젝트 추가
4. 커스터마이즈 패널 연결
5. 메시에 컬러/파츠 반영

## 9. 추천 결론

가장 먼저 붙일 만한 커스터마이즈는:

- 헤어
- 상의
- 소품

입니다.

이 세 가지만 있어도 캐릭터 차이가 확실히 보입니다.

지금 단계에서는

`거울 오브젝트 -> 아바타 꾸미기 패널 -> 헤어/상의/소품 변경`

부터 시작하는 것이 가장 효율적입니다.
