# Secret Base UX Concept

## 1. 방향 전환

이 앱은 더 이상 "채팅 앱에 부가 기능이 붙은 형태"로 가기보다,

`사람들이 함께 운영하는 아지트 앱`

으로 재정의하는 편이 맞습니다.

핵심 변화는 단순합니다.

- 기존의 대화방 리스트를 `아지트 리스트`로 바꾼다
- 메시지는 아지트 안의 한 기능이 된다
- 아지트는 친구 중심일 수도 있고 가족 중심일 수도 있다
- 가족 아지트는 대화보다 `생활 운영`이 더 중요하다
- AI 모델, 모델 선택, AI 채팅은 현재 범위에서 완전히 제외한다

즉, "채팅방"이 기본 단위가 아니라 "아지트"가 기본 단위가 됩니다.

## 2. OurSecretBase에서 가져올 점

참고한 파일:

- `C:\workspace\OurSecretBase\lib\screens\home_screen.dart`
- `C:\workspace\OurSecretBase\lib\widgets\hideout_card.dart`
- `C:\workspace\OurSecretBase\lib\widgets\custom_app_bar.dart`
- `C:\workspace\OurSecretBase\lib\theme\app_theme.dart`

가져올 점:

- 홈의 중심을 대화가 아니라 아지트 목록으로 두는 구조
- 따뜻하고 친근한 색감, 둥근 카드, 낮은 진입장벽
- 아지트 카드에서 멤버 수와 분위기를 바로 보여주는 패턴
- 상단 액션은 단순하게 두고, 본문에서 핵심 행동을 하게 만드는 구조

가져오지 않을 점:

- `model_selection_screen.dart`
- `ai_debug_screen.dart`
- `chat_screen.dart`의 AI 중심 진입 흐름

이번 컨셉에서는 AI는 제품 정의에서 제외합니다.

## 3. 새 제품 정의

### 한 줄 정의

가족과 친구가 함께 머무르고, 일정과 해야 할 일을 운영하고, 필요한 대화를 이어가는 공유 아지트 앱

### 핵심 오브젝트

`Hideout`

속성:

- 이름
- 타입: `friends` | `family`
- 커버 이미지 또는 테마
- 멤버
- 최근 활동
- 다음 일정
- 주요 상태 요약

중요한 점:

- 친구 아지트와 가족 아지트는 같은 엔티티지만 UX는 달라야 한다
- 같은 리스트에 보이되, 내부 정보 구조는 타입별로 달라져야 한다

## 4. 정보 구조

권장 하단 탭:

1. `아지트`
2. `사람들`
3. `일정`
4. `내 정보`

의미:

- `아지트`: 전체 허브. 기존 채팅 리스트가 이곳으로 바뀜
- `사람들`: 친구, 가족, 초대, 권한 관리
- `일정`: 모든 아지트의 일정과 할 일 통합 뷰
- `내 정보`: 프로필, 알림, 계정, 권한

현재 앱 기준 해석:

- `Chats` 탭은 `Hideouts` 탭으로 교체
- 기존 채팅 상세는 독립 1순위 화면이 아니라 아지트 상세 안의 한 섹션으로 이동
- `Friends` 탭은 유지하되, 초대와 관계 관리 중심으로 재정의

## 5. 아지트 리스트 UX

이 화면이 앱의 얼굴입니다.

### 화면 목적

- 내가 속한 모든 아지트를 한 번에 본다
- 어떤 아지트에 지금 들어가야 하는지 빠르게 판단한다
- 가족 아지트는 실용 정보, 친구 아지트는 분위기와 최근 활동을 강조한다

### 권장 구성

상단:

- 사용자 이름
- 오늘 요약 한 줄
- 알림 버튼
- 새 아지트 만들기 버튼

본문 상단:

- 필터 칩: `전체 / 가족 / 친구`
- 중요 요약 스트립:
  - 오늘 일정 몇 개
  - 처리 안 된 할 일 몇 개
  - 확인 필요한 위치 알림 몇 개

아지트 카드:

- 커버
- 아지트 이름
- 타입 배지: `가족` 또는 `친구`
- 멤버 아바타
- 최근 활동 한 줄
- 다음 일정 또는 해야 할 일 요약
- 상태 칩

가족 아지트 카드 예시:

- "민서 학교 도착"
- "오늘 가족 일정 3개"
- "미완료 할 일 2개"

친구 아지트 카드 예시:

- "주말 캠핑 플랜 업데이트"
- "사진 4장 추가됨"
- "새 메모 1개"

핵심 원칙:

- 카드만 봐도 지금 어디를 열어야 하는지 판단 가능해야 한다
- 가족 아지트는 유틸리티 중심, 친구 아지트는 분위기와 활동 중심으로 요약한다

## 6. 가족 아지트 UX

가족 아지트는 "대화방"이 아니라 "가족 운영 보드"에 가까워야 합니다.

### 기본 섹션

1. `오늘`
2. `위치`
3. `할 일`
4. `일정`
5. `대화`

### 첫 화면 권장 구조

상단 가족 상태 카드:

- 가족 이름
- 오늘의 핵심 상태
- 긴급 알림 유무

바로 아래 카드:

- 자녀 위치 상태
- 다음 이동 또는 귀가 시간
- 오늘의 할 일 완료율
- 오늘 가족 일정

하단 섹션:

- 가족 공지
- 최근 사진
- 짧은 대화 흐름

### 위치 UX

위치 기능은 친구 아지트에 노출하지 않습니다.

가족 아지트에서만:

- 자녀별 현재 상태: `학교`, `학원`, `이동 중`, `집`
- 마지막 확인 시각
- 배터리 또는 위치 갱신 상태
- 지도 전체 화면 진입

주의:

- 위치는 기본 공개가 아니라 보호자/허용된 구성원 기준으로 보여야 한다
- 카드에는 "정확한 좌표"보다 "현재 맥락"을 우선 표시하는 편이 UX가 좋다

예:

- "민서 · 학교 · 08:42 확인"
- "준호 · 이동 중 · 피아노 학원 가는 길"

### 할 일 / 시간표 UX

가족 아지트에는 아이별 카드가 필요합니다.

각 카드:

- 오늘 시간표
- 오늘 해야 할 일
- 완료 체크
- 보호자 메모

이 구조가 중요한 이유:

- 가족은 채팅보다 "오늘 뭘 해야 하지?"를 더 자주 확인한다
- 대화를 길게 읽는 대신 카드로 바로 행동할 수 있어야 한다

## 7. 친구 아지트 UX

친구 아지트는 가족 아지트보다 훨씬 가볍고 감성적이어야 합니다.

### 기본 섹션

1. `라운지`
2. `대화`
3. `플랜`
4. `앨범`
5. `멤버`

### 첫 화면 권장 구조

상단:

- 아지트 커버
- 아지트 이름
- 오늘의 메모 또는 분위기 한 줄

본문:

- 오늘 계획 카드
- 최근 사진 스트립
- 고정 메모
- 짧은 대화 타임라인

핵심 원칙:

- 친구 아지트는 채팅 로그보다 "우리만의 공간" 감각이 먼저 와야 한다
- 대화는 중요하지만 첫 인상은 커버, 메모, 플랜, 사진이 만든다

## 8. 생성 플로우

새 아지트를 만들 때 가장 먼저 묻는 것은 이름이 아니라 타입이어야 합니다.

### Step 1. 타입 선택

- `친구 아지트`
- `가족 아지트`

### Step 2. 멤버 선택

- 친구 또는 가족 구성원 초대
- 가족 아지트는 역할 지정:
  - 보호자
  - 자녀

### Step 3. 기본 테마 설정

- 커버 색감
- 아이콘
- 설명 한 줄

### Step 4. 초기 모듈 설정

친구 아지트:

- 대화
- 플랜
- 앨범

가족 아지트:

- 일정
- 할 일
- 위치
- 대화

중요:

- 모든 아지트에 모든 기능을 넣지 않는다
- 타입에 맞는 기본 모듈을 먼저 열고, 나머지는 나중에 확장한다

## 9. 권장 화면 흐름

### Flow A. 앱 진입

`아지트 리스트 -> 우선순위 높은 아지트 선택 -> 아지트 상세`

### Flow B. 가족 확인

`아지트 리스트 -> 가족 아지트 -> 오늘 / 위치 / 할 일 확인 -> 필요한 경우 대화`

### Flow C. 친구 모임 준비

`아지트 리스트 -> 친구 아지트 -> 플랜 카드 확인 -> 앨범/대화 확인`

이 흐름이 현재 구조와 다른 핵심은:

- 대화가 종착점이 아니라 보조 기능이라는 점
- 리스트의 목적이 "누가 말했나"가 아니라 "어느 아지트를 지금 봐야 하나"라는 점

## 10. 디자인 방향

`OurSecretBase`의 따뜻한 톤은 유지하되, 더 정리된 구조가 필요합니다.

권장 방향:

- 따뜻한 베이지, 세이지, 코랄, 잔잔한 네이비
- 큰 커버 카드
- 작은 상태 칩
- 둥근 카드, 낮은 그림자
- 지도/일정/할 일처럼 실용 정보도 감성 톤 안에 넣기

피해야 할 것:

- 메신저 앱처럼 텍스트 줄만 많은 리스트
- 가족 아지트를 설정 화면처럼 딱딱하게 만드는 것
- 친구 아지트를 게임 UI처럼 과하게 꾸미는 것

## 11. 1차 MVP 범위

가장 현실적인 1차 범위:

### 공통

- 아지트 리스트
- 친구 아지트 / 가족 아지트 타입 분기
- 아지트 상세 상단 요약
- 멤버 관리

### 친구 아지트

- 라운지
- 짧은 대화
- 플랜 카드
- 앨범 미리보기

### 가족 아지트

- 오늘 카드
- 자녀 상태 카드
- 할 일 체크
- 가족 일정 카드

### 제외

- AI 모델 선택
- AI 채팅
- 복잡한 게임형 커스터마이즈
- 실시간 협업 편집

## 13. 방 기반 게임형 비주얼 대안

참고 이미지:

- https://tse1.mm.bing.net/th/id/OIP.zmXK8AQC836JR1xKLDt7ZQHaE8?rs=1&pid=ImgDetMain&o=7&rm=3

이 방향은 리스트/대시보드 중심 UX와 별개로,

`아지트를 밝은 방 한 칸처럼 보여주고, 그 위에 게임 UI를 얹는 방식`

입니다.

이미지에서 가져온 핵심 톤:

- 화이트 톤 벽
- 머스타드 침구
- 큰 창과 부드러운 햇살
- 작은 가구와 아기자기한 정리감

이를 그대로 복제하지 않고, 동물의 숲처럼 단순화한 방 구조로 다시 만든 목업:

### A. 순수 방 장면

![게임형 방 장면](assets/mockups/animal-hideout-room-concept.svg)

### B. 게임 UI가 올라간 방 장면

![게임형 방 UI 장면](assets/mockups/animal-hideout-room-ui-concept.svg)

이 대안이 잘 맞는 경우:

- 친구 아지트를 더 감성적이고 놀이처럼 보이게 하고 싶을 때
- 라운지 첫 화면에 "우리 공간" 감각을 강하게 주고 싶을 때
- 카드형 리스트보다 방 한 칸의 인상이 더 중요할 때

주의할 점:

- 가족 아지트 전체를 이 스타일로 밀면 실용 정보가 묻힐 수 있다
- 그래서 가족 아지트는 대시보드형, 친구 아지트는 방형 비주얼을 섞는 하이브리드가 더 적합하다

## 14. 구현 해석

현재 `Our Hangout` 코드 기준으로는 이 순서가 가장 자연스럽습니다.

1. `Chats`를 `Hideouts`로 교체
2. 기존 room row를 hideout card 구조로 변경
3. hideout type을 `friends | family`로 추가
4. 가족 아지트 상세에 `오늘 / 위치 / 할 일 / 일정 / 대화` 섹션 추가
5. 친구 아지트 상세에 `라운지 / 대화 / 플랜 / 앨범` 섹션 추가
6. AI 관련 메뉴와 상태는 전부 제거한 상태로 유지

핵심은 채팅 기능을 버리는 게 아니라,

`채팅의 위치를 중심에서 아지트 안쪽으로 옮기는 것`

입니다.
